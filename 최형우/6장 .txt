6장 

RAM : CPU가 실행하고 싶은 프로그램이 저장되는 휘발성 저장 장치

- DRAM : 동적램. 일정 주기로 데이터 재활성화 필요, but 저전력 저렴 고집적 장점 多
- SRAM : 정적램. DRAM과 정 반대 -> 주로 사용되는건 DRAM. 캐시 메모리에 사용
- SDRAM : 클럭 신호 동기화 DRAM
- DDR SDRAM : 대역폭을 넓힌 SDRAM. DDR이 2배, DDR2가 4배, DDR4가 16배...  최근 가장 많이 사용



물리주소 : 메모리가 사용하는 하드웨어 상 주소
논리 주소 : 프로그램이 사용하는 0번부터 시작하는 주소

메모리 관리장치(MMU) : 물리 <-> 논리 주소간의 변환. MMU안의 베이스 레지스터에 저장된 값에 CPU가 준 논리주소값을 더해주는 식
한계 레지스터 : 다른 프로그램의 물리주소에 침범하지 않게 주소의 최대 크기를 가지고 있음. 그런 상황 발생시 인터럽트 발생


저장 장치 계층 구조 : CPU에 가까울수록 빠르고, 용량 작고, 비싸다.

캐시 메모리 : CPU가 메모리에 접근 덜 하도록 자주 사용하는 정보를 저장해 놓는 SRAM 기반의 저장장치
	- 캐시 적중률 : 캐시 메모리 내의 데이터가 CPU에 활용되는 비율(=캐시 히트 횟수/ (캐시 히트 횟수 + 캐시 미스 횟수))
	- 참조 지역성의 원리 : 캐시 메모리에 담을 데이터를 결정하는 원칙.
		1) 시간 지역성 : 최근에 접근했던 메모리 공간에 다시 접근하려는 경향
		2) 공간 지역성 : 접근한 메모리 공간 근처를 접근하려는 경향



7장
하드디스크 : 둥그런 플래터를 스핀들로 돌리며 디스크 암에 달린 헤드로 정보를 읽고 씀. 
	- 플래터 : 하나의 동심원을 트랙이라 부르고 이는 작은 섹터들로 나눠져있음. 여려개 곂쳐진 트랙을 합친게 실린더
	- 탐색시간 : 원하는 트랙까지 헤드를 옮기는 시간
	- 회전지연 : 헤드 위치까지 플래터를 회전시키는 시간

플래시 메모리 : 셀 < 페이지 < 블록 < 플레인 < 다이의 형태로 구성
	- 셀 :  SLC, MLC, TLC가 있음. 우측으로 갈수록 수명 짧고 느리지만 싸다
 	- 페이지 : Free(텅 빈 상태), Valid(유효한 데이터 있는 상태), Invalid(쓰레기 데이터 있는 상태) 상태로 구분가능
		- 가비지 컬렉션 : 유효한 페이지들만 다른 블록으로 복사한뒤, 기존 블록 삭제하는 식으로 해서 용량 정리 가능

RAID : 데이터의 안전성 or 성능을 높이기 위해 여러 보조기억장치를 하나의 기억장치처럼 사용하는 기술. 여러 레이드 레벨이 있음
	- RAID0 : 단순히 데이터를 번갈아 가면서 저장. 나눠진 하드디스크의 갯수배만큼 읽고 쓰는게 빠름 
	- RAID1 : 미러링된 복사본을 만드는 구조. RAID0보단 느리지만 안전성 높음
	- RAID4 : 완전한 복사본이 아니고, 오류 검출&복구를 위한 패리티 비트를 따로 한군데에 저장.
	- RAID5 : RAID4에서 일어나는 병목현상을 줄이기 위해, 패리티 비트를 골고루 분배해 둔 방식
	- RAID6 : RAID5와 유사하나, 패리티를 두군데에 저장하는 방식. RAID5보다 느리다 더욱 안전
